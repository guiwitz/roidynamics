Multiple rois¶

If you have multiple rois in your roi file, you can use it almost out of the box like in the regular case. Here we reproduce the analysis seen in the first part and highlight places where you can explicitly use roi options:

import matplotlib.pyplot as plt
import numpy as np
import skimage.io

import matplotlib
cmap = matplotlib.colors.ListedColormap(np.random.rand(256,3))

from microfilm.dataset import MultipageTIFF
import microfilm.microplot
from splitmask import splitmask

image_path = '../splitmask/tests/test_folders/test_multipage_good/'
roi_path = '../splitmask//tests/test_folders/test_multipage_good/multiroi.bmp'

image = MultipageTIFF(image_path)
roi = skimage.io.imread(roi_path)

cm = splitmask.get_roi_cm(roi_im=roi)
im_np = image.load_frame(channel_name=image.channel_name[1], frame=20)

Defining sectors¶

Defining sectors is done as previously. The construction of sector masks also is identical:

sector_width = 5
num_sectors = 5

radius_labels = splitmask.create_concentric_mask(center=cm, im_dims=image.dims,
                                                         sector_width=sector_width, num_sectors=num_sectors)
colors_rad, cmap_rad = splitmask.get_cmap_labels(radius_labels, cmap_name='cool')

radius_labels.shape

(4, 196, 171)

The differences is that now the mask containing the sectors contains multiple planes, one for each roi. We can show it in composite color mode and also show the index of each roi:

cmaps = ['pure_green', 'pure_blue', 'pure_red', 'pure_cyan']
microfilm.microplot.microshow(radius_labels, cmaps=cmaps);

The same goes for the angular sectors:

angular_width = 20
max_rad =25
sector_labels = splitmask.create_sector_mask(center=cm, im_dims=image.dims,
                                                     angular_width=angular_width, max_rad=max_rad)

microfilm.microplot.microshow(sector_labels, cmaps=cmaps);

Plotting multiple roi sectors¶

We can here again use the plot_sectors function. If we use it as before, all rois are just shown with a default map:

splitmask.plot_sectors(image, radius_labels, time=10, channel=image.channel_name[1]);

However, we can also explicitly specify:

  • cmap colormaps to use for each roi

  • alpha transparency to apply

  • display the index of each roi in the image title with show_labels

cmaps = ['pure_green', 'pure_blue', 'pure_red', 'viridis']
splitmask.plot_sectors(image, radius_labels, channel=image.channel_name[1],
                       im_cmap='Greys', cmap=cmaps, alpha=0.6, show_labels=True);

Finally, we can also show single rois using the roi parameter:

splitmask.plot_sectors(image, radius_labels, channel=image.channel_name[1], roi=2,
                       im_cmap='Greys', cmap=cmaps, alpha=0.6, show_labels=True);

Changing roi order¶

With the above plots, knowing which roi corresponds to which index, we can also reorder rois if necessary. Of course this has to be done before signal extraction as otherwise there will be a mismatch between roi position and measurement. Let’s plot again the figure:

cmaps = ['pure_green', 'pure_blue', 'pure_red', 'viridis']
splitmask.plot_sectors(image, radius_labels, channel=image.channel_name[1],
                       im_cmap='Greys', cmap=cmaps, alpha=0.6, show_labels=True);

Now we define a new order for our four rois, e.g. [1,3,2,0] where for example the 1 means that the second roi (1 in blue above) should now be the first roi and therefore be colored in green:

radius_labels_reordered = splitmask.roi_reorder(sectors=radius_labels, new_order=[1,3,2,0])

cmaps = ['pure_green', 'pure_blue', 'pure_red', 'viridis']
splitmask.plot_sectors(image, radius_labels_reordered, channel=image.channel_name[1],
                       im_cmap='Greys', cmap=cmaps, alpha=0.6, show_labels=True);

Extracting intensities¶

Extracting intensities also works as previously. Only now the output contains multiple rois instead of just one:

channel = [image.channel_name[0], image.channel_name[1]]
signal_radius = splitmask.measure_intensities(time_image=image, channels=channel, 
                                                      im_labels=radius_labels,max_time=51)

    When we plot the results, here also we have the additional option of specifying which roi we want to use via the roi parameter:

    myplot = splitmask.plot_signals(signal_radius, channel=0, roi=1, color_array=colors_rad);

    Saving the labels¶

    Finally, when we save the extracted signals, each intensity is also asisgned a roi index. In order to know which index corresponds to which roi, you can save the labelled roi image:

    splitmask.save_labelled_roi('labelled_roi2.tif', roi_im=roi)

    Don’t forget the if you re-order rois, the original labelled roi above is not correct. In that case, you should save an image of the re-ordered rois with labels.
